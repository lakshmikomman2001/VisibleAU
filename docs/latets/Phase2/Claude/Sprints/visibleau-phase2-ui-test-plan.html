Cross-cutting — test on every Phase 2 page

locked build rules
Brand-isolation gate security
Every brand-scoped page must enforce assertBrandAccess(user, brandId) — org-scoped RLS alone does not enforce per-brand access. Test that user A cannot load user B's brand pages (expect 403/404, never the data).
Tier-gating & locked teasers tier
Paid screens render for the right tier; lower tiers see the correct locked teaser, not the data. Discovery (Agency+), Reports (Growth+ generation), Competitive (Agency+) are the clearest cases.
Explainability contract
Score-bearing pages must show {rationale, confidence_label, confidence_note, top_action} — never a bare number. Rationale must be specific (>30 chars, naming a real signal), the same honesty discipline as the methodology fix.
Score scale: 0–100, not 0–10
Composite & dimension scores are 0–100 (the scale the Phase 1 sample-result fix corrected). mention_source_ratio is 0–1. Don't let any page mislabel "/10" or show NaN where a div-by-zero guard should return NULL.
Status-enum correctness
Four separate enums, never unified: audits='complete' · workflow_runs='completed' · remediation_tasks (…/wont_fix, not 'done') · content_drafts (draft/approved/published/rejected). Watch labels on Workflow & Autopilot.
Accessibility & responsive (RM-02)
Dynamic-status regions (generating→ready, toasts) in aria-live="polite" with aria-busy; responsive breakpoints incl. mobile <375px (no horizontal scroll). Forward build rule on every screen.
Logged-in only · brand context
All under app/(auth)/ — test signed in. Brand-scoped pages need a real brand with audit data (use Bondi Plumbing). A brand with no audits should show an empty state, not a crash.
